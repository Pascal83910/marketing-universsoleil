## 1. PHILOSOPHIE ET PRINCIPES FONDATEURS

### 1.1 Ce qu'est la recherche marketing

La recherche marketing est la démarche systématique de collecte, d'analyse et d'interprétation d'informations sur un marché, des clients, des concurrents ou un contexte, dans le but d'éclairer une décision commerciale ou stratégique.

Elle répond à une question fondamentale : **"Sur quoi repose cette décision — une intuition ou une preuve ?"**

### 1.2 Ce que la recherche marketing n'est pas

| Confusion fréquente | Réalité |
|---------------------|---------|
| "Faire de la recherche = faire un sondage" | Un sondage n'est qu'une méthode parmi une douzaine |
| "Les données parlent d'elles-mêmes" | Les données ne deviennent des insights qu'après interprétation |
| "La recherche ralentit la décision" | Une mauvaise décision coûte plus cher que 2 semaines de recherche |
| "On connaît déjà nos clients" | L'intuition est un point de départ, jamais une conclusion |
| "Plus de données = meilleure décision" | La pertinence de la donnée prime sur le volume |

### 1.3 Les 5 principes du framework

```
┌─────────────────────────────────────────────────────────────────┐
│  1. QUESTION D'ABORD  │  Toute recherche commence par une       │
│                       │  question précise, pas par une méthode  │
├─────────────────────────────────────────────────────────────────┤
│  2. FIT MÉTHODE/OBJET │  La méthode est choisie après la        │
│                       │  question, jamais avant                 │
├─────────────────────────────────────────────────────────────────┤
│  3. TRIANGULATION     │  Toute conclusion majeure repose sur    │
│                       │  au moins 2 sources convergentes        │
├─────────────────────────────────────────────────────────────────┤
│  4. ACTIONNABILITÉ    │  Un insight sans décision associée      │
│                       │  n'est qu'une information               │
├─────────────────────────────────────────────────────────────────┤
│  5. ITÉRATION         │  La recherche est un cycle, pas un      │
│                       │  projet linéaire à sens unique          │
└─────────────────────────────────────────────────────────────────┘
```

## 3. PHASE 1 — CADRAGE : DÉFINIR LE PROBLÈME À RÉSOUDRE

> *"Un problème bien posé est à moitié résolu."* — Charles Kettering

### 3.1 Pourquoi cette phase est la plus critique

La majorité des recherches marketing qui échouent (coûteuses, inutilisées, non actionnables) le font non pas en Phase 3 ou 4, mais ici. On collecte des données pour répondre à la mauvaise question — ou à une question floue.

**Test : votre question est-elle assez précise ?**

| Question floue ❌ | Question précise ✅ |
|------------------|-------------------|
| "Que pensent nos clients ?" | "Pourquoi le taux de réachat a-t-il baissé de 12 % sur le segment 35–45 ans au T3 ?" |
| "Quel est le marché ?" | "Quelle est la taille du marché adressable pour notre offre B2B dans le secteur santé en France ?" |
| "Est-ce que notre nouveau produit va marcher ?" | "Quelle caractéristique de notre prototype est la plus déterminante dans l'intention d'achat ?" |
| "Qui sont nos concurrents ?" | "Quels sont les 3 concurrents qui captent nos prospects perdus et sur quel argument ?" |

### 3.2 L'outil de cadrage — Le Brief de Recherche

Avant toute collecte, rédiger un **Brief de Recherche** qui répond à ces 8 questions :

```
╔══════════════════════════════════════════════════════╗
║              BRIEF DE RECHERCHE                      ║
╠══════════════════════════════════════════════════════╣
║ 1. DÉCISION À PRENDRE                               ║
║    Quelle décision concrète cette recherche         ║
║    doit-elle éclairer ?                             ║
║    → Ex : "Lancer ou non la gamme premium en 2026"  ║
╠══════════════════════════════════════════════════════╣
║ 2. QUESTION(S) DE RECHERCHE                         ║
║    Qu'est-ce qu'on cherche à comprendre,            ║
║    mesurer ou découvrir ?                           ║
║    → 1 question principale + 2–3 sous-questions max ║
╠══════════════════════════════════════════════════════╣
║ 3. HYPOTHÈSES DE DÉPART                             ║
║    Qu'est-ce qu'on croit savoir déjà ?             ║
║    → Rendre explicites les présupposés              ║
╠══════════════════════════════════════════════════════╣
║ 4. PÉRIMÈTRE                                        ║
║    Marché / Segment / Géographie / Période          ║
║    In scope vs out of scope                         ║
╠══════════════════════════════════════════════════════╣
║ 5. UTILISATEURS DES RÉSULTATS                       ║
║    Qui va lire et utiliser les conclusions ?        ║
║    → Direction, équipe produit, commercial ?        ║
╠══════════════════════════════════════════════════════╣
║ 6. CONTRAINTES                                      ║
║    Budget / Temps / Accès aux données               ║
║    Ressources humaines disponibles                  ║
╠══════════════════════════════════════════════════════╣
║ 7. CRITÈRE DE SUCCÈS                                ║
║    Comment sait-on que la recherche a répondu       ║
║    à la question ? Quel niveau de confiance ?       ║
╠══════════════════════════════════════════════════════╣
║ 8. DATE DE LIVRAISON                                ║
║    Deadline non négociable pour la décision         ║
╚══════════════════════════════════════════════════════╝
```

### 3.3 Identifier le type de question

La nature de la question détermine tout le reste du processus.

| Type | Question caractéristique | Nature | Méthodes typiques |
|------|--------------------------|--------|------------------|
| **Exploratoire** | "Que se passe-t-il ?" "Pourquoi ?" | Qualitatif | Entretiens, ethnographie, netnographie |
| **Descriptive** | "Combien ?" "Qui ?" "Quand ?" | Quantitatif | Sondages, données panel, statistiques |
| **Causale** | "Est-ce que X cause Y ?" | Expérimental | A/B test, test de concept, expérimentation |
| **Prédictive** | "Que va-t-il se passer si ?" | Modélisation | Analyse de cohortes, scoring, modèles |

### 3.4 Validation du cadrage — Les 5 questions de contrôle

Avant de passer à la Phase 2, répondre à ces 5 questions :

1. **Si on trouvait la réponse demain, changerait-elle quelque chose ?** (sinon, la recherche est inutile)
2. **Le décideur est-il prêt à agir en fonction des résultats ?** (sinon, la recherche ne sera pas utilisée)
3. **La question est-elle trop large pour être traitée avec les ressources disponibles ?** (sinon, la scoper)
4. **Existe-t-il déjà une réponse dans des données existantes ?** (commencer toujours par les sources secondaires)
5. **La réponse est-elle éthiquement et légalement accessible ?** (RGPD, données sensibles, accords de confidentialité)

## 4. PHASE 2 — DESIGN : CHOISIR LA BONNE MÉTHODE

### 4.1 La boussole Qualitatif / Quantitatif

```
                  QUALITATIF
                 (comprendre)
                      │
          Explorer    │    Expliquer
          Découvrir   │    Contextualiser
                      │
EXPLORATOIRE ─────────┼───────── CONFIRMATOIRE
                      │
          Mesurer     │    Tester
          Quantifier  │    Généraliser
                      │
                 QUANTITATIF
                  (mesurer)
```

**Règle de décision :**
- Je ne sais pas ce que je ne sais pas → **Qualitatif d'abord**
- Je veux confirmer ou mesurer quelque chose → **Quantitatif**
- Je veux comprendre ET mesurer → **Mixte (qual puis quant)**

### 4.2 Arbre de décision méthodologique

```
La question est-elle connue et définie ?
│
├── NON → Recherche exploratoire
│         → Entretiens individuels
│         → Focus groups
│         → Observation terrain
│         → Analyse de données non structurées
│
└── OUI → La réponse doit-elle être généralisable ?
          │
          ├── NON → Méthodes qualitatives confirmatives
          │         → Entretiens ciblés
          │         → Analyse documentaire
          │
          └── OUI → Le phénomène est-il mesurable directement ?
                    │
                    ├── OUI → Méthodes quantitatives
                    │         → Sondage (n > 100)
                    │         → Panel, données internes
                    │         → Analyse statistique
                    │
                    └── INDIRECTEMENT → Méthodes expérimentales
                                        → A/B test
                                        → Test de concept
                                        → Expérimentation contrôlée
```

### 4.3 Matrice de sélection des méthodes

| Question type | Budget faible | Budget moyen | Budget élevé |
|--------------|--------------|-------------|-------------|
| Comprendre les motivations | Entretiens internes (clients existants) | Entretiens qualitatifs externes (10–15) | Étude ethnographique |
| Mesurer la satisfaction | Questionnaire court (NPS) | Sondage en ligne (200–500 répondants) | Panel longitudinal |
| Évaluer un nouveau concept | Test de landing page | Test de concept qualitatif | Test quantitatif + qualitatif |
| Analyser la concurrence | Desk research, prix publics | Mystery shopping + analyse digitale | Étude de marché Xerfi/Kantar |
| Comprendre le parcours client | Cartographie interne | Entretiens + analytics web | Ethnographie + eye tracking |
| Segmenter le marché | Analyse de la base CRM | Sondage segmentation (500+) | Étude panel nationale |

### 4.4 Planifier la recherche — Le Plan de Recherche

| Élément | À définir |
|---------|-----------|
| Méthode(s) sélectionnée(s) | Qualitative / Quantitative / Mixte |
| Population cible | Qui doit-on interroger / observer ? |
| Taille de l'échantillon | Combien de personnes / sources ? |
| Critères de sélection | Quels filtres de recrutement ? |
| Outil de collecte | Guide d'entretien / Questionnaire / Grille |
| Canal de collecte | Téléphone / Face-à-face / En ligne / Données |
| Calendrier | Dates de collecte, jalons intermédiaires |
| Responsables | Qui fait quoi ? |
| Budget | Coûts prévus par poste |
| Livrables attendus | Rapport / Présentation / Dashboard |

## 5. PHASE 3 — COLLECTE : ALLER CHERCHER LA DONNÉE

### 5.1 Hiérarchie des sources — Commencer par ce qui existe déjà

**Règle d'or :** Avant de collecter de nouvelles données, épuiser les sources existantes. La recherche primaire est coûteuse ; la recherche secondaire est souvent suffisante pour 60–70 % des questions.

```
PYRAMIDE DES SOURCES DE DONNÉES

         ┌───────────────────────┐
         │   DONNÉES PRIMAIRES   │  ← Collectées spécifiquement
         │  (cher, lent, précis) │    pour cette recherche
         └──────────┬────────────┘
                    │
         ┌──────────▼────────────┐
         │   DONNÉES INTERNES    │  ← CRM, ventes, analytics,
         │  (gratuit, rapide,    │    tickets SAV, avis clients
         │   sous-exploité)      │
         └──────────┬────────────┘
                    │
         ┌──────────▼────────────┐
         │   DONNÉES SECONDAIRES │  ← Études de marché, rapports
         │   EXTERNES            │    sectoriels, INSEE, presse
         │  (accessible, variés) │    spécialisée, publications
         └───────────────────────┘
                  ↑
           Commencer ici
```

### 5.2 Sources secondaires internes — L'or caché

| Source | Type de données | Usage typique |
|--------|----------------|--------------|
| CRM | Profil clients, historique achats, fréquence | Segmentation, LTV, attrition |
| Google Analytics / Analytics web | Comportement navigation, sources trafic | Parcours digital, intention |
| Tickets SAV / Support | Problèmes récurrents, verbatims | Irritants, besoins non couverts |
| Avis clients (Google, Trustpilot) | Verbatims spontanés | Motivations, irritants |
| Emails et appels entrants | Verbatims non sollicités | Questions fréquentes, objections |
| Données de vente | Volume, panier, mix produit | Tendances, saisonnalité |
| Taux de conversion | Entonnoirs, abandon | Frictions dans le parcours |
| NPS historique | Satisfaction longitudinale | Évolution de la satisfaction |

### 5.4 Standards de collecte primaire

#### Entretiens qualitatifs — Guide de conduite

```
STRUCTURE D'UN ENTRETIEN (60–90 min)

[0–5 min]   INTRODUCTION
            - Présentation, objectif de l'entretien
            - Règles : pas de bonne/mauvaise réponse
            - Accord d'enregistrement (RGPD)

[5–15 min]  CONTEXTE ET PROFIL
            - Comprendre la personne avant le sujet
            - Ses habitudes, son rôle, son environnement

[15–60 min] CORPS DE L'ENTRETIEN
            - Questions ouvertes → questions de relance
            - Entonnoir : général → spécifique
            - Techniques : ZMET, incident critique, tri de cartes

[60–75 min] EXPLORATION APPROFONDIE
            - "Pouvez-vous me raconter une situation concrète ?"
            - "Qu'est-ce qui vous a surpris ?"
            - "Qu'est-ce qui vous a déçu ?"

[75–90 min] CLÔTURE
            - "Y a-t-il quelque chose d'important que je n'ai pas demandé ?"
            - Remerciements, suite du projet
```

**Règles d'or pour les entretiens :**
- Toujours enregistrer (avec accord) et transcrire, jamais prendre des notes seules
- Le silence est un outil : ne pas remplir les blancs
- Jamais valider ou invalider les réponses
- Poser "pourquoi ?" 3 fois si nécessaire (méthode des "5 pourquoi")
- Taille minimale pour saturation : **8 à 12 entretiens** par segment homogène

#### Sondages quantitatifs — Principes de conception

**Structure d'un questionnaire :**
```
[SECTION 1] Qualification et sélection (2–3 questions)
            → Filtrer les non-cibles avant de commencer

[SECTION 2] Questions comportementales (5–10 questions)
            → Ce que la personne fait réellement
            → "Avez-vous acheté X dans les 6 derniers mois ?"

[SECTION 3] Questions attitudinales (5–15 questions)
            → Ce que la personne pense et ressent
            → Échelles de Likert (1–5 ou 1–7), NPS, sémantique

[SECTION 4] Questions sur le sujet principal (5–15 questions)
            → Le cœur de la recherche

[SECTION 5] Données socio-démographiques (3–5 questions)
            → Âge, CSP, géographie — toujours en dernier
```

**Règles de construction des questions :**

| À faire ✅ | À éviter ❌ |
|-----------|-----------|
| Questions neutres et équilibrées | Questions orientées ("Ne pensez-vous pas que...") |
| Une seule idée par question | Questions doubles ("Êtes-vous satisfait de la qualité ET du prix ?") |
| Vocabulaire du répondant | Jargon interne ou technique |
| Ordre logique et progressif | Ruptures thématiques brusques |
| Pré-tester sur 5 personnes | Lancer sans test pilote |
| Options exhaustives et mutuellement exclusives | Listes incomplètes ou chevauchantes |

**Tailles d'échantillon indicatives :**

| Population cible | Marge d'erreur souhaitée | Taille minimale |
|-----------------|-------------------------|----------------|
| Marché de masse (> 1M) | ± 5 % (niveau de confiance 95 %) | **385 répondants** |
| Marché de masse | ± 3 % | 1 067 répondants |
| Segment spécifique | ± 5 % | 385 répondants |
| B2B niché | Indicatif | **50–100 répondants** |
| Qualitatif exploratoire | Saturation théorique | **8–15 entretiens** |

### 5.5 Protocole de qualité de la collecte

**Avant la collecte :**
- [ ] Le guide / questionnaire a été testé sur 3–5 personnes
- [ ] Les critères de sélection de l'échantillon sont documentés
- [ ] Le consentement RGPD est prévu (données personnelles)
- [ ] Le calendrier et les responsabilités sont assignés

**Pendant la collecte :**
- [ ] Vérification quotidienne de la qualité (données aberrantes, taux de complétion)
- [ ] Alerte si le taux de réponse est < 50 % (sondage) ou si la saturation n'est pas atteinte (qualitatif)
- [ ] Documentation des incidents ou biais détectés

**Après la collecte :**
- [ ] Nettoyage des données (doublons, réponses incohérentes, temps de complétion anormal)
- [ ] Vérification de la représentativité de l'échantillon vs population cible
- [ ] Archivage des données brutes et de la documentation de collecte

## 6. PHASE 4 — ANALYSE : TRANSFORMER LA DONNÉE EN INSIGHT

### 6.1 La distinction cruciale : donnée vs information vs insight

```
DONNÉES BRUTES
"73 % des répondants ont cité 'le prix' comme frein"
        ↓
INFORMATION STRUCTURÉE
"Le frein prix est plus fort chez les 35–50 ans (-12 pts vs 25–35 ans)
 et chez les clients ayant déjà subi une panne"
        ↓
INSIGHT ACTIONNABLE
"Le prix n'est pas un frein absolu : c'est une expression de méfiance
 envers la durabilité chez les clients ayant eu une mauvaise expérience.
 → Lever ce frein passe par une garantie étendue visible, pas par une
   réduction de prix"
```

**Un insight répond toujours à "So what ?" (Et alors ?) et "What now ?" (Maintenant quoi ?)**

### 6.2 Analyse qualitative — Méthode de codage

**Processus en 4 étapes :**

1. **Transcription** : retranscrire tous les entretiens mot à mot (ou utiliser un outil IA de transcription)
2. **Première lecture** : lire sans annoter, se laisser imprégner
3. **Codage ouvert** : annoter chaque passage avec un code thématique (ex. "frein_prix", "confiance_marque", "experience_SAV")
4. **Codage axial** : regrouper les codes en thèmes supérieurs, identifier les relations entre thèmes

**Matrice d'analyse qualitative :**

| Thème | Verbatims clés | Fréquence | Intensité | Segment |
|-------|---------------|-----------|----------|---------|
| Confiance / Peur de la panne | "J'ai peur que ça tombe en panne après la garantie" | 9/12 | Forte | 50+ ans |
| Prix perçu comme élevé | "C'est cher mais si c'est fiable..." | 7/12 | Moyenne | Tous |
| Besoin d'autonomie | "Je veux ne plus dépendre d'EDF" | 11/12 | Très forte | 40–65 ans |

### 6.3 Analyse quantitative — Séquence analytique

```
NIVEAU 1 — STATISTIQUES DESCRIPTIVES (toujours en premier)
├── Moyennes, médianes, distributions
├── Fréquences et pourcentages
└── Identification des valeurs aberrantes

NIVEAU 2 — ANALYSES CROISÉES
├── Tableaux croisés (variable X par segment)
├── Tests de significativité (Chi², t-test)
└── Corrélations entre variables

NIVEAU 3 — ANALYSES MULTIVARIÉES (si pertinent)
├── Segmentation (cluster analysis)
├── Analyse factorielle (réduction de dimensions)
├── Régression (prédiction)
└── Conjoint analysis (préférences produit)

NIVEAU 4 — VISUALISATION ET NARRATION
├── Graphiques adaptés au type de données
├── Hiérarchisation des résultats (important → secondaire)
└── Formulation des insights
```

### 6.4 Choisir le bon graphique

| Type de données | Objectif | Graphique recommandé |
|----------------|---------|---------------------|
| 1 variable quantitative | Distribution | Histogramme, boîte à moustaches |
| Comparaison entre groupes | Classement | Barres horizontales triées |
| Évolution dans le temps | Tendance | Courbe (line chart) |
| Part d'un tout | Composition | Barres empilées (pas camembert > 5 items) |
| Relation entre 2 variables | Corrélation | Nuage de points (scatter plot) |
| Données géographiques | Répartition spatiale | Carte choroplèthe |
| Verbatims qualitatifs | Fréquence des termes | Nuage de mots (avec précaution) |

### 6.5 Identifier les insights — Les 5 types

| Type d'insight | Définition | Exemple |
|---------------|-----------|---------|
| **Insight de tension** | Contradiction entre ce que le client dit et ce qu'il fait | "Je dis chercher le prix bas mais je choisis toujours la marque premium" |
| **Insight de besoin non comblé** | Un désir exprimé ou latent sans solution satisfaisante | "Je veux savoir en temps réel ce que produit mon installation" |
| **Insight de perception** | Écart entre la réalité et la perception qu'en a le client | "La chaudière granulés est perçue comme compliquée mais est en réalité plus automatique que le gaz" |
| **Insight de contexte** | Le contexte change fondamentalement l'usage ou la décision | "La décision est prise à deux mais c'est souvent l'un des conjoints qui mène les recherches seul" |
| **Insight de déclencheur** | L'événement qui fait basculer de l'intention à l'achat | "La réception d'une facture anormalement élevée déclenche 70 % des prises de contact" |

## 7. PHASE 5 — SYNTHÈSE : COMMUNIQUER LES RÉSULTATS

### 7.1 La règle de la pyramide de Minto

Les résultats se communiquent du plus important vers le détail, jamais l'inverse.

```
┌─────────────────────────────────────────────────────┐
│            RÉPONSE À LA QUESTION                    │  ← En 1 phrase, d'emblée
│         "Notre cible prioritaire est X car Y"       │
├─────────────────────────────────────────────────────┤
│       ARGUMENT 1    │  ARGUMENT 2  │  ARGUMENT 3    │  ← 3 preuves qui soutiennent
├─────────────────────────────────────────────────────┤
│  Données  │ Verbatim │  Stats  │  Données  │  Etc.  │  ← Détails en soutien
└─────────────────────────────────────────────────────┘
```

**Ce que les décideurs veulent d'abord :** la conclusion. Les détails viennent valider, pas précéder.

### 7.2 Structure d'un rapport de recherche

```
1. RÉSUMÉ EXÉCUTIF (1 page maximum)
   → La question / La méthode / Les 3 insights clés / Les recommandations

2. CONTEXTE ET OBJECTIFS
   → La décision à prendre / La question de recherche / Le périmètre

3. MÉTHODOLOGIE
   → Comment les données ont été collectées / Taille d'échantillon /
     Dates / Limites et biais identifiés

4. RÉSULTATS
   → Organisés par question de recherche, pas par méthode
   → Chaque section = 1 insight titre + données + implications

5. CONCLUSIONS ET RECOMMANDATIONS
   → "Ce que nous avons appris" + "Ce que nous recommandons de faire"
   → Priorisées par impact / faisabilité

6. ANNEXES
   → Guide d'entretien / Questionnaire / Tableaux bruts / Verbatims étendus
```

### 7.3 Formater les insights pour maximiser l'impact

**Template d'un insight bien formulé :**

```
TITRE (affirmation, pas titre neutre)
"Les clients seniors se décident en 3 rendez-vous, pas en 1"

PREUVE (données qui le démontrent)
→ 8 entretiens sur 12 mentionnent plusieurs échanges avant décision
→ Analytics : 73 % des leads > 55 ans font ≥ 3 visites du site

CONTEXTE (pourquoi c'est vrai)
→ L'investissement est perçu comme risqué : ils veulent être rassurés,
  pas vendus

IMPLICATION (ce que ça change)
→ Réviser le processus commercial : ne pas forcer la signature au 1er RDV,
  planifier un 2e appel de réassurance J+7

ACTION RECOMMANDÉE (concrète et assignable)
→ Créer un email de suivi J+7 pour ce segment avec témoignages clients
  similaires / Responsable : commercial / Délai : 3 semaines
```

### 7.4 Adapter la communication à l'audience

| Audience | Format | Durée | Niveau de détail |
|---------|--------|-------|-----------------|
| Direction générale | Présentation executive | 15–20 min | Top insights + recommandations |
| Équipe marketing | Workshop de restitution | 60–90 min | Insights + implications opérationnelles |
| Équipe commerciale | Fiche de synthèse | 5 min de lecture | Insights terrain + scripts adaptés |
| Équipe produit | Rapport complet | 30–60 min de lecture | Insights + personas + parcours |
| External stakeholders | Rapport public ou one-pager | Variable | Adapté selon accord confidentialité |

## 8. PHASE 6 — ACTION : MESURER L'IMPACT

### 8.1 Boucler la boucle — Recherche → Décision → Mesure

Une recherche marketing non suivie d'action est un coût sans valeur. Une action non mesurée ne permet pas d'apprendre.

```
[INSIGHT] → [RECOMMANDATION] → [DÉCISION] → [ACTION] → [MESURE] → [NOUVEAU INSIGHT]
                                                              ↑
                                                       Retour Phase 1
```

### 8.2 Template de plan d'action post-recherche

| Recommandation | Décision prise | Action concrète | Responsable | Délai | KPI de mesure | Résultat attendu |
|---------------|---------------|----------------|------------|-------|--------------|-----------------|
| Exemple : "Ajouter témoignages vidéo sur page produit" | Oui / Non / En attente | Contacter 3 clients satisfaits pour tourner des vidéos | [Nom] | 6 semaines | Taux de conversion page produit | +15 % |

### 8.3 Évaluer le ROI de la recherche

**À 6 mois, répondre à :**
1. La décision a-t-elle été prise sur la base des insights ? (utilisation)
2. Les actions recommandées ont-elles été mises en œuvre ? (adoption)
3. Les KPIs ciblés ont-ils évolué dans le sens prévu ? (impact)
4. Quelles hypothèses de la recherche se sont avérées incorrectes ? (apprentissage)

## 9. BOÎTE À OUTILS — MÉTHODES DE RECHERCHE DÉTAILLÉES

### 9.1 Tableau des méthodes — Vue d'ensemble

| Méthode | Type | Question adressée | Avantages | Limites | Coût | Délai |
|---------|------|------------------|----------|---------|------|-------|
| **Entretien individuel** | Qualitatif | Pourquoi ? Qu'est-ce que ? | Profondeur, nuance | Non généralisable | Moyen | 2–4 semaines |
| **Focus group** | Qualitatif | Quelles perceptions partagées ? | Interaction, idées collectives | Conformité sociale, facilitation | Moyen | 2–3 semaines |
| **Observation terrain** | Qualitatif | Comment ? Dans quel contexte ? | Comportements réels | Temps, interprétation | Élevé | 3–6 semaines |
| **Netnographie** | Qualitatif | Que disent-ils en ligne ? | Verbatims spontanés, gratuité | Échantillon non contrôlé | Faible | 1–2 semaines |
| **Sondage en ligne** | Quantitatif | Combien ? Qui ? | Rapidité, coût, échelle | Superficialité, biais | Faible | 1–3 semaines |
| **Sondage téléphonique** | Quantitatif | Idem + profils difficiles | Taux de réponse élevé | Coût, biais intervieweur | Élevé | 2–4 semaines |
| **Analyse de données CRM** | Quantitatif | Qui achète quoi, quand ? | Données comportementales réelles | Données passées seulement | Faible | 1 semaine |
| **Web analytics** | Quantitatif | Comment naviguent-ils ? | Comportement en temps réel | Anonymat, sans "pourquoi" | Faible | En continu |
| **A/B test** | Expérimental | Quelle version performe mieux ? | Causalité prouvée | Trafic requis, contexte contrôlé | Faible | 2–8 semaines |
| **Test de concept** | Mixte | Est-ce que ce concept intéresse ? | Évaluation avant développement | Intention ≠ comportement | Moyen | 2–4 semaines |
| **Analyse concurrentielle** | Secondaire | Que font les concurrents ? | Rapidité, coût, exhaustivité | Données passées, publiques | Faible | 1–2 semaines |
| **Analyse des avis clients** | Secondaire | Quels problèmes / satisfactions ? | Verbatims authentiques | Biais de représentation | Faible | 1 semaine |
| **Mystery shopping** | Observation | Comment se passe l'expérience ? | Réaliste, comparatif | Subjectivité, légalité | Moyen | 2–4 semaines |
| **Tri par cartes (card sorting)** | Qualitatif | Comment catégorisent-ils ? | Structure mentale réelle | Technique, formation | Faible | 1–2 semaines |
| **Test utilisateur (UX)** | Mixte | Comment utilisent-ils un produit ? | Comportement réel, frictions | Recrutement, facilitation | Moyen | 2–3 semaines |

### 9.2 La netnographie — Écoute des conversations en ligne

**Principe :** Observer ce que les clients disent spontanément sur les forums, réseaux sociaux, avis en ligne, groupes Facebook, Reddit, etc., sans les solliciter.

**Sources à scruter :**
- Avis Google, Trustpilot, G2, Capterra
- Groupes Facebook thématiques
- Sous-reddits pertinents
- Forums spécialisés (questions, plaintes, recommandations)
- Commentaires LinkedIn et posts sectoriels
- YouTube : commentaires sous vidéos de concurrents

**Grille d'analyse netnographique :**

| Verbatim | Source | Date | Sujet | Sentiment | Insight potentiel |
|---------|--------|------|-------|----------|-----------------|
| "J'ai attendu 3 semaines pour qu'on me rappelle..." | Google Avis | Fév 2026 | SAV / Délai | Négatif | Irritant n°1 = délai de réponse |

### 9.3 L'analyse concurrentielle — Cadre structuré

**Dimensions à analyser :**

```
1. POSITIONNEMENT
   → Proposition de valeur / Promesse / Territoire de marque
   → Segments cibles (déclarés ou déduits)
   → Avantages concurrentiels mis en avant

2. OFFRE PRODUIT / SERVICE
   → Gamme, prix, packaging
   → Fonctionnalités distinctives
   → Conditions commerciales (garanties, SAV, délais)

3. PRÉSENCE DIGITALE
   → Site web : trafic estimé, SEO, contenu
   → Réseaux sociaux : canaux, fréquence, engagement
   → Publicité : messages, ciblage (via Meta Ad Library, Google Ads)

4. PERCEPTION CLIENT
   → Avis et notes (Google, Trustpilot, sectoriels)
   → Verbatims : ce qu'ils adorent / ce qu'ils détestent

5. FORCES DE VENTE ET DISTRIBUTION
   → Canaux de distribution
   → Modèle de vente (direct, indirect, omnicanal)

6. DYNAMIQUE FINANCIÈRE
   → CA et croissance (si disponibles — SIREN, Infogreffe)
   → Investissements, levées de fonds

7. INNOVATION ET R&D
   → Lancements récents, brevets
   → Signaux faibles d'évolution de l'offre
```

**Livrable recommandé :** Matrice concurrentielle (1 ligne = concurrent, colonnes = dimensions analysées, notation de 1 à 5)

## 10. GESTION DES BIAIS ET CONTRÔLE QUALITÉ

### 10.1 Les 10 biais les plus fréquents en recherche marketing

| Biais | Description | Comment le limiter |
|-------|-------------|-------------------|
| **Biais de désirabilité sociale** | Le répondant dit ce qu'il pense être "bien" | Questions comportementales > attitudinales. Anonymat. |
| **Biais de confirmation** | On cherche des données qui confirment l'hypothèse | Définir les critères d'invalidation avant la collecte |
| **Biais de recrutement** | L'échantillon n'est pas représentatif | Définir des quotas. Diversifier les sources. |
| **Biais d'acquiescement** | Les répondants disent "oui" à tout | Formuler certaines questions en sens opposé |
| **Biais de l'interviewer** | L'enquêteur influence les réponses | Scripts neutres. Double blind quand possible. |
| **Biais de mémorisation** | Les personnes ne se souviennent pas avec précision | Courtes périodes de référence. Journaux de bord. |
| **Biais de primauté/récence** | On se souvient du début et de la fin, pas du milieu | Rotation des items dans les questionnaires |
| **Biais de halo** | Une impression générale teinte toutes les évaluations | Séparer les évaluations de différents aspects |
| **Survivorship bias** | On étudie seulement les succès ou les clients actuels | Inclure les clients perdus, les non-clients |
| **Biais de projection** | L'équipe de recherche projette ses propres opinions | Triangulation avec des évaluateurs indépendants |

### 10.2 Checklist de qualité — Avant de livrer

**Sur les données :**
- [ ] L'échantillon est représentatif de la population cible
- [ ] Les données aberrantes ont été identifiées et traitées
- [ ] Les biais connus sont documentés et mentionnés dans le rapport
- [ ] La marge d'erreur est calculée et communiquée

**Sur l'analyse :**
- [ ] Les conclusions sont étayées par des données, pas des impressions
- [ ] Les corrélations ne sont pas présentées comme des causalités
- [ ] Les résultats négatifs ou contre-intuitifs sont inclus
- [ ] Les limites de la recherche sont explicitement énoncées

**Sur la communication :**
- [ ] La réponse à la question initiale est clairement formulée
- [ ] Chaque recommandation est actionnable et assignable
- [ ] Le niveau de confiance des conclusions est indiqué
- [ ] Les données sources sont archivées et accessibles

## 11. ADAPTER LE FRAMEWORK À LA TAILLE DE L'ENTREPRISE

### 11.1 TPE / Indépendant (1–10 personnes)

**Contraintes :** Budget faible, peu de temps, pas d'équipe dédiée.

**Version allégée du framework :**

```
PHASE 1 (30 min) : Brief en 3 questions
  → Quelle décision ? Quelle question ? Quel délai ?

PHASE 2 (1h) : Choix d'une seule méthode
  → Priorité : sources secondaires gratuites + 5 entretiens clients

PHASE 3 (3–5 jours) : Collecte ciblée
  → 5 entretiens clients (30 min chacun) + analyse avis Google
  → Desk research (1 journée)

PHASE 4 (1 jour) : Synthèse simple
  → 3 insights clés sous forme de Post-its

PHASE 5 (1h) : Communication orale
  → Partage avec l'équipe, décision prise

PHASE 6 : Mesure d'un seul KPI
  → "Est-ce que ça a fonctionné ?"
```

**Outils gratuits recommandés :**
- Google Forms (questionnaires)
- Notion ou Obsidian (notes d'entretiens)
- Google Analytics (web analytics)
- Typeform (formulaires premium)
- Answer the Public (SEO et intentions de recherche)

### 11.2 PME (10–250 personnes)

**Contraintes :** Budget modéré, 1–2 personnes marketing, temps limité mais processus possibles.

**Ce qui change :**
- Investir dans un outil CRM si pas encore en place
- Systématiser la collecte d'avis après chaque vente
- Budget études : 3 000 – 15 000 €/an pour 2–3 études thématiques
- Externaliser la collecte (agence ou freelance) et garder l'analyse en interne

### 11.3 ETI / Grand groupe (250+ personnes)

**Ce qui change :**
- Direction Études et Market Intelligence dédiée
- Abonnements panels (Nielsen, Kantar, GfK)
- Protocoles de recherche standardisés
- Comité de validation des conclusions
- Centralisation des données dans un data warehouse

## 13. GLOSSAIRE

| Terme | Définition |
|-------|-----------|
| **Brief de recherche** | Document initial qui cadre la question, les objectifs et les contraintes d'une recherche |
| **Biais cognitif** | Erreur systématique dans le traitement de l'information qui fausse les conclusions |
| **Codage** | Processus d'étiquetage des données qualitatives par thèmes pour faciliter l'analyse |
| **Insight** | Compréhension profonde et non évidente d'un comportement ou d'une motivation, qui génère une action |
| **Marge d'erreur** | Intervalle de confiance autour d'un résultat quantitatif, lié à la taille de l'échantillon |
| **Méthode mixte** | Combinaison de méthodes qualitatives et quantitatives dans une même recherche |
| **Netnographie** | Observation et analyse des comportements et discours en ligne |
| **Persona** | Représentation fictive et archétypale d'un segment de clients, basée sur des données réelles |
| **Plan de recherche** | Document opérationnel détaillant les méthodes, l'échantillon, le calendrier et les livrables |
| **Population cible** | L'ensemble des individus ou entités sur lesquels porte la recherche |
| **Primaire (donnée)** | Donnée collectée spécifiquement pour cette recherche (entretien, sondage) |
| **Profondeur vs étendue** | Trade-off entre comprendre peu de personnes en profondeur (qualitatif) ou beaucoup superficiellement (quantitatif) |
| **Recherche exploratoire** | Recherche menée quand on ne sait pas encore ce qu'on cherche — pour découvrir |
| **RGPD** | Règlement Général sur la Protection des Données — cadre légal européen sur les données personnelles |
| **Saturation théorique** | Point où collecter de nouvelles données n'apporte plus d'information nouvelle — arrêt de la collecte qualitative |
| **Secondaire (donnée)** | Donnée existante, collectée par un tiers pour un autre usage |
| **Taille d'échantillon** | Nombre d'individus nécessaires pour que les résultats soient représentatifs ou statistiquement significatifs |
| **Triangulation** | Validation d'une conclusion par la convergence d'au moins deux sources ou méthodes différentes |
| **Verbatim** | Citation mot à mot d'un répondant, utilisée pour illustrer un insight qualitatif |
